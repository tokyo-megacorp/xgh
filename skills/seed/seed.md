# xgh:seed — Seed xgh Context into Secondary AI Tools

Seeds xgh project context into every secondary AI coding tool detected in the current environment. Each tool gets a tailored context file at the path it expects.

> **Output format:** Follow the [xgh output style guide](../../templates/output-style.md). Start with `## 🐴🤖 xgh seed`. Use a markdown table for results. End with an italicized next step.

---

## Step 1: Locate the plugin cache

Find the xgh plugin cache to get the context template:

```bash
PLUGIN_DIR=$(find ~/.claude/plugins/cache -path "*/xgh/*/skills/seed/seed.md" -print -quit 2>/dev/null | sed 's|/skills/seed/seed.md||')
```

The context source files live at:
- `.agents/skills/xgh/context.md` (relative to repo root) — for agent-style tools (Codex, OpenCode)
- `.gemini/skills/xgh/context.md` (relative to repo root) — for Gemini CLI

If `PLUGIN_DIR` is empty, the plugin is likely running from the source repo itself — use the current working directory as the base.

---

## Step 2: Detect installed tools

Run these checks in a single bash block:

```bash
# Binary checks
CODEX_FOUND=$(command -v codex >/dev/null 2>&1 && echo true || echo false)
GEMINI_FOUND=$(command -v gemini >/dev/null 2>&1 && echo true || echo false)
OPENCODE_FOUND=$(command -v opencode >/dev/null 2>&1 && echo true || echo false)
AIDER_FOUND=$(command -v aider >/dev/null 2>&1 && echo true || echo false)

# Directory checks
CURSOR_FOUND=$([ -d ".cursor" ] && echo true || echo false)
CONTINUE_FOUND=$([ -d ".continue" ] && echo true || echo false)
```

---

## Step 3: Build the context content

Generate project context by reading the current repo state. The content is the same for all tools but may be formatted differently. Use the following template — fill in real values from the current repo:

```
# xgh Project Context
_Auto-generated by /xgh-seed on <ISO date>. Do not edit manually._

## Project Summary
<Read config/project.yaml and extract: name, tagline, description. If not available, use git remote + repo name.>

## Key Decisions
<Read the most recent 5-10 entries from .xgh/context-tree/ if it exists. Otherwise omit this section.>

## Architecture
- **Plugin manifest**: `.claude-plugin/plugin.json` at repo root
- **Source layout**: `skills/`, `commands/`, `agents/`, `config/`, `hooks/`, `templates/`
- **Static instructions**: `templates/xgh-instructions.md` → copied to `.xgh/xgh.md`
- **Context tree**: `.xgh/context-tree/` — decisions, architecture, conventions knowledge base
- **Memory tools**: `magi_store`, `magi_query`, `magi_navigate`, `magi_status` (MAGI MCP)
- **Plugin cache**: `~/.claude/plugins/cache/extreme-go-horse/xgh/<version>/`
- **Seeded tool dirs**: `.gemini/skills/xgh/` (Gemini CLI), `.agents/skills/xgh/` (Agents)

## Workflow Conventions
- Push a plan before implementing (plan-PR-review-merge loop)
- Skill files live in `skills/<name>/<name>.md`; commands in `commands/<name>.md`
- All generated artifacts go to `.xgh/` (plans, specs, architecture decisions)

## Active Branch
- **Branch**: <run: git branch --show-current>
- **Focus**: <read first line of most recent .xgh/plans/*.md if available>

## Recent Commits
<run: git log --oneline -5 and format as bullet list>

---
_Refresh: run `/xgh-seed` in Claude Code_
```

Use `git log --oneline -5` and `git branch --show-current` to populate the dynamic sections. Read `config/project.yaml` for project name and tagline. If `.xgh/context-tree/` exists, read its most recent files to populate Key Decisions.

---

## Step 4: Seed each detected tool

For each tool marked as found in Step 2, write the context file to the correct path. Create parent directories as needed.

### Codex (`CODEX_FOUND=true`)

Target: `.agents/skills/xgh/context.md`

```bash
mkdir -p .agents/skills/xgh
```

Write the context content (Step 3) to `.agents/skills/xgh/context.md`.

### Gemini (`GEMINI_FOUND=true`)

Target: `.gemini/skills/xgh/context.md`

```bash
mkdir -p .gemini/skills/xgh
```

Write the context content (Step 3) to `.gemini/skills/xgh/context.md`.

### OpenCode (`OPENCODE_FOUND=true`)

Target: `.opencode/skills/xgh/context.md`

```bash
mkdir -p .opencode/skills/xgh
```

Write the context content (Step 3) to `.opencode/skills/xgh/context.md`.

### Cursor (`CURSOR_FOUND=true`)

Target: `.cursor/rules/xgh.md`

Cursor rules have a different format — concise rules pointing to canonical files, not a full context dump.

Write the following to `.cursor/rules/xgh.md` (replacing existing file):

```markdown
# Cursor Rules — xgh (eXtreme Go Horse)

Full agent instructions: see `AGENTS.md` at the repository root.

## Project type

<read config/project.yaml tech_stack field, or default to: "Bash/YAML/Markdown plugin">

## Must-follow rules

- Start every task by reading `AGENTS.md`
- Write failing tests in `tests/` before implementing (bash `assert_*` pattern)
- All bash scripts: `#!/usr/bin/env bash` + `set -euo pipefail`
- Track progress with `- [x]` checkboxes in `.xgh/plans/`
- Never commit secrets — env vars only

## Install

```bash
claude plugin install xgh@extreme-go-horse
/xgh-init
```

## Test commands

```bash
bash tests/test-config.sh
```

## Key paths

- `skills/` — Claude Code skills (xgh:* namespace)
- `commands/` — slash commands (/xgh-*)
- `config/` — project.yaml, agents.yaml, team.yaml, workflow.yaml
- `.xgh/plans/` — implementation plans
- `tests/` — bash test suite
```

### Aider (`AIDER_FOUND=true`)

Target: `.aider/prompts/xgh.md`

```bash
mkdir -p .aider/prompts
```

Aider reads prompt files as system-level context. Write the context content (Step 3) to `.aider/prompts/xgh.md`.

### Continue (`CONTINUE_FOUND=true`)

Target: `.continue/rules/xgh.md`

```bash
mkdir -p .continue/rules
```

Continue extension reads rules from `.continue/rules/`. Write the context content (Step 3) to `.continue/rules/xgh.md`.

---

## Step 5: Output the results table

After seeding all detected tools, output:

```
## 🐴🤖 xgh seed

| Tool | Binary / Dir | Target | Status |
|------|-------------|--------|--------|
| Codex | `codex` | `.agents/skills/xgh/context.md` | ✅ seeded |
| Gemini | `gemini` | `.gemini/skills/xgh/context.md` | ✅ seeded |
| OpenCode | `opencode` | `.opencode/skills/xgh/context.md` | ⏭️ not installed |
| Cursor | `.cursor/` | `.cursor/rules/xgh.md` | ✅ seeded |
| Aider | `aider` | `.aider/prompts/xgh.md` | ⏭️ not installed |
| Continue | `.continue/` | `.continue/rules/xgh.md` | ⏭️ not installed |
```

Use ✅ for seeded, ⏭️ for not detected (not an error — tool not installed).

If no tools were detected at all, output:

```
No secondary AI tools detected in the current environment.

Install any of: codex, gemini, opencode, aider — or create a `.cursor/` or `.continue/` directory.
```

End with an italicized suggestion:

```
_Run `/xgh-seed` again after installing new AI tools to keep all contexts in sync._
```

---

## Error handling

- **Write permission denied**: Report `❌ <tool>: permission denied writing to <path>` and continue seeding remaining tools.
- **git command fails** (not a git repo): Omit "Active Branch" and "Recent Commits" sections from the context content; proceed normally.
- **config/project.yaml missing**: Use the git repo name as project name, omit tagline. Do not fail.
- **Context tree directory empty or missing**: Omit the "Key Decisions" section silently.

---

## Composability

This skill is called by `xgh:init` as part of Step 7a (after AGENTS.md generation). It can also be invoked standalone at any time:

- After installing a new AI tool in the environment
- After significant architectural decisions (to update context across all tools)
- As part of project onboarding for a new team member
