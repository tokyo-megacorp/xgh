# Provider-Based Retrieval Architecture

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the Claude-dependent CronCreate retrieval pipeline with a provider framework where each data source has its own fetch mechanism — bash scripts for open-auth services, MCP tool calls for OAuth-gated corporate services — generated automatically by `/xgh-track`.

**Architecture:** Each provider ships as a `spec.md` in the plugin — instructions Claude reads during `/xgh-track` to generate provider artifacts at `~/.xgh/providers/<name>/`. Providers operate in two modes: `bash` (self-contained `fetch.sh`, no Claude needed) or `mcp` (single-turn MCP tool call via CronCreate, uses existing OAuth). A discovery-based `retrieve-all.sh` orchestrator runs bash-mode providers. A separate lightweight CronCreate prompt handles mcp-mode providers. The analyze step remains Claude-powered but reads pre-fetched inbox files.

**Tech Stack:** Bash (`fetch.sh` scripts), `gh` CLI (GitHub), `curl` + `jq` (REST APIs), YAML (provider config), Markdown (provider specs as Claude instructions)

## Provider Contract

Every provider **must** satisfy this interface:

### Mode

Each provider operates in one of two modes:

| Mode | Auth | Executor | Turns | When to use |
|------|------|----------|-------|-------------|
| `bash` | CLI tool or API token in `tokens.env` | `retrieve-all.sh` (no Claude) | 1 | Open APIs, personal tokens, CLI-auth'd tools |
| `mcp` | OAuth via configured MCP server | CronCreate prompt (Claude) | 2-3 | Corporate SSO/OAuth-gated services where standalone tokens aren't available |

### `fetch.sh` (bash mode only)

- **Inputs:** sources `~/.xgh/tokens.env` for auth; reads `provider.yaml` for project config; reads `cursor` for last-seen state
- **Outputs:** writes `.md` files to `~/.xgh/inbox/` following the inbox item format; updates `cursor`; exits 0 on success, 1 on failure
- **Constraints:** no Claude dependency, no MCP calls, pure bash + curl/jq/gh

### `provider.yaml`

```yaml
name: github
version: 1
mode: bash                  # bash | mcp
auth:
  type: cli                 # cli | bearer | basic | api_key | mcp_oauth | none
  tool: gh                  # CLI tool name (for type: cli)
  env_var: null             # token env var name (for token-based auth)
  env_var_2: null           # second env var (for basic auth: user + token)
  mcp_server: null          # MCP server name (for type: mcp_oauth, e.g. "slack", "atlassian")
sources:                    # populated from ingest.yaml during /xgh-track
  - project: xgh
    repo: ipedro/xgh
    watch: [pull_requests, issues, releases]
  - project: inspector
    repo: ipedro/Inspector
    watch: [issues, pull_requests, releases]
cursor:
  strategy: timestamp       # timestamp | id | page_token
  format: iso8601           # iso8601 | unix | snowflake
inbox:
  source_type: github_pr    # prefix for inbox filenames
  urgency_keywords:         # simple keyword-based scoring
    - critical
    - blocker
    - P0
    - hotfix
    - security
# MCP mode fields (only when mode: mcp)
mcp:
  tools:                    # MCP tool(s) to call — Claude reads these at fetch time
    - name: mcp__slack__conversations_history
      params_template:      # ${VAR} placeholders filled from sources + cursor
        channel: "${SOURCE_ID}"
        limit: 50
        oldest: "${CURSOR}"
  result_mapping:           # jq-style path to extract items from MCP response
    items: ".messages"
    timestamp: ".ts"
    text: ".text"
    author: ".user"
```

### `cursor`

Plain text file. Single value: the last-seen timestamp, ID, or page token. Updated atomically by `fetch.sh` (bash mode) or by the MCP retrieval prompt (mcp mode).

### Inbox item format (unchanged)

```markdown
---
type: inbox_item
source_type: github_pr
source_repo: ipedro/xgh
source_ts: 2026-03-20T10:00:00Z
project: xgh
urgency_score: 30
processed: false
awaiting_direction: null
links_followed: []
---

PR #42: Fix authentication timeout
Author: contributor
Updated: 2026-03-20T10:00:00Z
Status: open
Review: CHANGES_REQUESTED
```

This is the **instruction document** Claude reads when scaffolding an unknown provider via `/xgh-track add-provider <name>`. It must contain:

1. **Questions to ask the user** (6 questions): provider name, base API URL, auth method ("Do you have an API token, or does this service authenticate only via MCP/OAuth?"), token env var name OR MCP server name, notifications/recent-activity endpoint pattern (or MCP tool name for mcp mode), cursor strategy (timestamp/id/page_token)
2. **Mode decision logic:**
   - User has API token or CLI tool → `mode: bash`

3. **`provider.yaml` generation instructions** — fill the schema from user answers, including `mode` and `mcp` section if applicable
4. **`fetch.sh` generation instructions** (bash mode only) — template with:
   - `#!/usr/bin/env bash` + `set -euo pipefail`
   - Source `~/.xgh/tokens.env`
   - Read `provider.yaml` with a simple parser (grep/awk, no python dependency)
   - Read cursor file, default to "24 hours ago" if missing
   - `curl` call to the API with auth header
   - `jq` to extract items
   - Loop: write each item as inbox `.md` file
   - Update cursor atomically (`tmp` + `mv`)
   - Log to `~/.xgh/logs/provider-<name>.log`
5. **MCP fetch instructions** (mcp mode only) — template for the `mcp` section in `provider.yaml`: tool names, parameter templates with `${SOURCE_ID}` and `${CURSOR}` placeholders, result mapping
6. **`tokens.env` instructions** (bash mode only) — tell user to add their token
7. **Connection test** — `curl` command (bash) or MCP tool call (mcp) to verify auth works
8. **Urgency keyword defaults** — ask user for domain-specific keywords

       text: ".text"
       author: ".user"

Note: macOS does not ship `timeout` by default. Use a portable bash function instead.

```bash
#!/usr/bin/env bash
set -euo pipefail

# retrieve-all.sh — Discovery-based provider orchestrator
# Finds and runs all mode:bash fetch.sh scripts in ~/.xgh/providers/
# Skips mode:mcp providers (handled by separate CronCreate prompt)
# Called by CronCreate every 5 minutes (1 Bash turn, no Claude)

PROVIDERS_DIR="${XGH_PROVIDERS_DIR:-$HOME/.xgh/providers}"
INBOX_DIR="$HOME/.xgh/inbox"
LOG_FILE="$HOME/.xgh/logs/retriever.log"
PAUSE_FILE="$HOME/.xgh/scheduler-paused"

# Portable timeout: use gtimeout (brew coreutils) or timeout if available, else skip
run_with_timeout() {
    local secs=$1; shift
    if command -v gtimeout &>/dev/null; then
        gtimeout "$secs" "$@"
    elif command -v timeout &>/dev/null; then
        timeout "$secs" "$@"
    else
        "$@"
    fi
}

# Guard: check pause file
if [ -f "$PAUSE_FILE" ]; then
    echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) retriever: paused" >> "$LOG_FILE"
    exit 0
fi

# Guard: check inbox dir exists
mkdir -p "$INBOX_DIR" "$HOME/.xgh/logs"

# Discover and run providers
total=0
success=0
failed=0
items_before=$(find "$INBOX_DIR" -name "*.md" -not -name "WARN_*" | wc -l | tr -d ' ')

for provider_dir in "$PROVIDERS_DIR"/*/; do
    [ -d "$provider_dir" ] || continue
    name=$(basename "$provider_dir")
    script="$provider_dir/fetch.sh"

    # Skip MCP-mode providers (no fetch.sh — handled by MCP CronCreate prompt)
    if grep -q "^mode: mcp" "$provider_dir/provider.yaml" 2>/dev/null; then
        continue
    fi

    if [ ! -x "$script" ]; then
        echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) retriever: WARN $name — fetch.sh not found or not executable" >> "$LOG_FILE"
        continue
    fi

    total=$((total + 1))

    rc=0
    run_with_timeout 30 bash "$script" 2>>"$HOME/.xgh/logs/provider-$name.log" || rc=$?
    if [ "$rc" -eq 0 ]; then
        success=$((success + 1))
    else
        failed=$((failed + 1))
        echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) retriever: ERROR $name — exit code $rc" >> "$LOG_FILE"
    fi
done

items_after=$(find "$INBOX_DIR" -name "*.md" -not -name "WARN_*" | wc -l | tr -d ' ')
new_items=$((items_after - items_before))

echo "$(date -u +%Y-%m-%dT%H:%M:%SZ) retriever: $total providers, $success ok, $failed failed, $new_items new items" >> "$LOG_FILE"
```
