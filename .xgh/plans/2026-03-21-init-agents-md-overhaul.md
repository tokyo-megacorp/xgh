## Task 2: Add Step 7a — Generate AGENTS.md + Wire Platform Files

**Files:**
- Modify: `skills/init/init.md`

- [ ] **Step 1: Insert Step 7a between Step 7 and Step 7b**

  Add the following section (full prose below). Position: after `If **no** (or user skips): Continue to Step 7b.` in Step 7.

  ```markdown
  ---

  ## Step 7a — Generate AGENTS.md + Wire Platform Files

  Always run this step. It creates (or regenerates) the canonical `AGENTS.md` for the
  current repository and wires all AI platform files to point to it.

  ### 7a.1 — Check existing AGENTS.md

  ```bash
  head -1 AGENTS.md 2>/dev/null || echo "MISSING"
  ```

  - **Missing** → generate fresh (continue to 7a.2).
  - **Has xgh-managed marker** (`<!-- AUTO-GENERATED by /xgh-init`) → re-scan and rewrite silently.
  - **Exists without marker** → ask:
    ```
    An AGENTS.md already exists without an xgh marker.
    Replace it with a generated one? [y/N — default: keep existing, only wire platform files]
    ```

  ### 7a.2 — Architecture scan

  Read the following to understand the project:
  - `README.md` (or `README.rst` / `README.adoc`) — project overview, install, usage
  - First package manifest found: `package.json`, `Cargo.toml`, `go.mod`, `pyproject.toml`, `Gemfile`, `build.gradle`
  - Top-level directory listing (`ls -1`) — infer repo layout
  - `.xgh/context-tree/` entries if present — use validated architecture docs

  ### 7a.3 — Generate AGENTS.md

  Write `AGENTS.md` at the repo root with these sections:

  ```markdown
  <!-- AUTO-GENERATED by /xgh-init — canonical instructions for all AI platforms. -->
  <!-- To regenerate: /xgh-init --regenerate-agents -->

  # AGENTS.md — <project name>

  > Canonical agent instructions. All platform files (CLAUDE.md, GEMINI.md,
  > .github/copilot-instructions.md) point here.

  ## What is <project>?
  <1-3 sentences from README>

  ## Tech Stack
  | Layer | Technology |
  |-------|-----------|
  <derived from manifest + README>

  ## Repo Layout
  | Directory | Purpose |
  |-----------|---------|
  <top-level dirs>

  ## Dev Commands
  | Command | What it does |
  |---------|-------------|
  | `<install>` | Install dependencies |
  | `<test>`    | Run tests |
  | `<build>`   | Build |
  <from README / package.json scripts / Makefile>

  ## Architecture Notes
  <key subsystems, entrypoints, data flow — 3-7 bullets from README + context tree>

  ## Conventions
  <coding style, naming, test discipline, PR expectations — from README / CONTRIBUTING / context tree>

  ## Environment
  <required env vars, secrets policy — from README / .env.example>

  ## Common Pitfalls
  <generated files, migrations, do-not-edit paths — from README + context tree gotchas>
  ```

  Only add optional sections if clearly present:
  - **Agent Roster** — if `agents/` directory or `config/agents.yaml` exists
  - **Automation Map** — if `config/triggers.yaml` + `config/workflows/` exist
  - **Release / Deploy** — if CI config or release scripts exist

  ### 7a.4 — Wire platform files

  Write the following files (create or overwrite):

  **`CLAUDE.md`** (local only — gitignored):
  ```markdown
  <!-- AUTO-GENERATED by /xgh-init. Edit AGENTS.md instead. -->
  @AGENTS.md
  ```

  **`GEMINI.md`** (local only — gitignored):
  ```markdown
  <!-- AUTO-GENERATED by /xgh-init. Edit AGENTS.md instead. -->
  @AGENTS.md
  ```
  *(If `@` include syntax is unsupported by the detected Gemini version, use:)*
  ```markdown
  <!-- AUTO-GENERATED by /xgh-init. Edit AGENTS.md instead. -->
  Read AGENTS.md at the repository root before making changes to this project.
  ```

  **`.github/copilot-instructions.md`** (committed — create if missing, update if xgh-managed):
  ```markdown
  # Copilot Instructions

  > Full agent instructions: see [`AGENTS.md`](../AGENTS.md) at the repository root.

  Read `AGENTS.md` before making any changes to this repository.
  ```

  ### 7a.5 — Gitignore additions

  Add to `.gitignore` if not already present:
  ```
  CLAUDE.md
  GEMINI.md
  ```
  Do **not** gitignore `AGENTS.md` or `.github/copilot-instructions.md`.

  ### 7a.6 — Offer /xgh-seed (if dispatch platforms detected)

  If Codex CLI (`command -v codex`), Gemini CLI (`command -v gemini`), or OpenCode (`command -v opencode`) are detected, ask:

  ```

  Seed AGENTS.md context into these platforms now? [Y/n]
  ```

  If yes → invoke `xgh:seed`.

  Continue to Step 7b.
  ```
